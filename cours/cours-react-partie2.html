useRef — Références DOM

useRef retourne un objet { current: valeur }. C'est un conteneur persistant que React ne surveille pas. Deux usages principaux :

  • Référence DOM : accéder directement à un élément HTML (focus, lecture de dimensions, animations)
  • Valeur sans re-rendu : stocker une info qui change mais ne doit pas déclencher de re-rendu (id de timer, flag booléen...)
  DIFFÉRENCE AVEC useState :

  ┌──────────────────────────────────┬──────────────────────────────────┐
  │          useState                │           useRef                 │
  ├──────────────────────────────────┼──────────────────────────────────┤
  │ Déclenche un re-rendu            │ Ne déclenche AUCUN re-rendu      │
  │ quand la valeur change           │ quand .current change            │
  ├──────────────────────────────────┼──────────────────────────────────┤
  │ La valeur s'affiche dans le JSX  │ La valeur est "en coulisses"     │
  ├──────────────────────────────────┼──────────────────────────────────┤
  │ Pour : données UI, compteurs,    │ Pour : références DOM, timers,   │
  │        formulaires...            │        valeurs de comparaison    │
  └──────────────────────────────────┴──────────────────────────────────┘

  RÉFÉRENCE DOM :

  const refInput = useRef(null)
                           │
  Au premier rendu :       └─ .current = null
                                         │
  Après le rendu :                       └─ .current = l'élément <input> du DOM

  <input ref={refInput} />
         ↑
  React connecte automatiquement refInput.current à cet élément
  après l'avoir inséré dans le DOM.
ChampRecherche.jsx — useRef pour focus et DOM 
import { useRef, useEffect } from 'react'

function ChampRecherche() {

  // useRef(null) crée { current: null }.
  // null car on n'a pas encore accès à l'élément DOM.
  // React mettra à jour .current après le premier rendu.
  const refInput = useRef(null)

  // Après le montage ([] = une fois), on donne le focus à l'input.
  // On accède à l'élément via refInput.current
  // qui pointe maintenant sur le vrai élément <input> du DOM.
  useEffect(() => {
    // refInput.current est l'élément <input> HTML.
    // .focus() est une méthode native du DOM, pas de React.
    refInput.current.focus()
  }, [])

  const effacerEtFocaliser = () => {
    // On peut lire et modifier les propriétés DOM directement.
    // Attention : modifier .value directement bypass le state React.
    // C'est acceptable ici car c'est une action one-shot.
    refInput.current.value = ''
    refInput.current.focus()
  }

  const lireDimensions = () => {
    // getBoundingClientRect() retourne la taille et position
    // de l'élément dans la page. Impossible sans référence DOM.
    const rect = refInput.current.getBoundingClientRect()
    console.log('Largeur de l\'input :', rect.width)
  }

  return (
    <div>
      {/* On lie la ref à l'élément avec l'attribut "ref". */}
      <input
        ref={refInput}
        type="text"
        placeholder="Rechercher un atelier..."
      />
      <button onClick={effacerEtFocaliser}>Effacer</button>
    </div>
  )
}

useMemo & useCallback — Performance

À chaque re-rendu, React ré-exécute toute la fonction du composant. Si cette fonction contient un calcul coûteux (filtrer 10 000 éléments) ou crée une fonction qui cause des re-rendus en cascade, ces deux hooks mémorisent le résultat pour éviter le recalcul inutile.

  useMemo — mémorise une VALEUR :

  const total = useMemo(
    () => elements.reduce((acc, e) => acc + e.prix, 0),
    [elements]          ← recalcule seulement si elements change
  )

  SANS useMemo : recalcule à CHAQUE re-rendu du composant.
  AVEC useMemo : recalcule SEULEMENT si [elements] change.


  useCallback — mémorise une FONCTION :

  const gererClic = useCallback(
    (id) => { sInscrire(id) },
    [sInscrire]         ← recrée seulement si sInscrire change
  )

  Pourquoi ? Chaque rendu RECRÉE les fonctions.
  Si tu passes une fonction recréée à un enfant React.memo,
  l'enfant re-render inutilement.
  useCallback évite ça en stabilisant la référence de la fonction.


  QUAND LES UTILISER ?

  useMemo    → calculs coûteux (filtres, tris, agrégations sur > 100 éléments)
  useCallback → fonctions passées en props à des composants React.memo
  À éviter   → sur tout le reste (la mémorisation coûte aussi de la mémoire)
TableauDeBord.jsx — useMemo pour le filtrage 
import { useState, useMemo } from 'react'
import { utiliserSkillHub } from '../contextes/ContexteSkillHub'
import CarteAtelier from '../composants/CarteAtelier'
import FiltresAteliers from '../composants/FiltresAteliers'

// Valeur initiale des filtres — définie HORS du composant
// pour éviter qu'elle soit recréée à chaque rendu.
const FILTRES_INITIAUX = { recherche: '', categorie: '', niveau: '' }

function TableauDeBord() {
  const { ateliers, utilisateur } = utiliserSkillHub()
  const [filtres, setFiltres] = useState(FILTRES_INITIAUX)

  // Gestionnaire générique pour tous les filtres.
  // "cle" est le nom du champ : 'recherche', 'categorie', ou 'niveau'.
  // On copie les filtres actuels (...filtres) et on écrase la clé concernée.
  const mettreAJourFiltre = (cle, valeur) => {
    setFiltres(prev => ({ ...prev, [cle]: valeur }))
  }

  // ─── useMemo ──────────────────────────────────────────────────
  // Sans useMemo : ce filtrage serait recalculé à CHAQUE rendu
  //               de TableauDeBord, même si ateliers et filtres
  //               n'ont pas changé (ex: si un parent re-render).
  //
  // Avec useMemo : le filtrage ne recalcule QUE SI
  //               ateliers ou filtres ont réellement changé.
  //
  // Syntaxe : useMemo(() => valeurCalculée, [dépendances])
  //           La fonction fléchée DOIT retourner la valeur.
  // ────────────────────────────────────────────────────────────
  const ateliersFiltres = useMemo(() => {
    return ateliers.filter(a => {

      // === '' signifie "filtre vide = pas de filtre appliqué"
      const okRecherche = filtres.recherche === ''
        || a.titre.toLowerCase().includes(filtres.recherche.toLowerCase())
        || a.tags.some(t => t.toLowerCase().includes(filtres.recherche.toLowerCase()))

      const okCategorie = filtres.categorie === ''
        || a.categorie === filtres.categorie

      const okNiveau = filtres.niveau === ''
        || a.niveau === filtres.niveau

      // L'atelier passe le filtre seulement si TOUS les critères sont ok
      return okRecherche && okCategorie && okNiveau
    })
  }, [ateliers, filtres]) // ← recalcule si ateliers ou filtres changent

  return (
    <div>
      <h1>Bonjour, { utilisateur.nom }</h1>
      <FiltresAteliers
        filtres={filtres}
        surChangement={mettreAJourFiltre}
        surReinit={() => setFiltres(FILTRES_INITIAUX)}
      />
      <div className="grille">
        { ateliersFiltres.map(a => (
          <CarteAtelier key={a.id} atelier={a} />
        )) }
      </div>
    </div>
  )
}

export default TableauDeBord

useReducer — État complexe

Quand tu as beaucoup de useState liés entre eux, ou quand les transitions d'état sont complexes (plusieurs cas possibles), useReducer organise tout dans une fonction reducer. Tu ne modifies plus le state directement — tu dispatches des actions qui décrivent "ce qui s'est passé", et le reducer calcule le nouvel état.

  PRINCIPE :

  Action envoyée               État actuel
  { type: 'INSCRIRE',          { ateliers: [...],
    id: 'workshop-1' }           utilisateur: {...} }
         │                              │
         │                              │
         └────────────┐ ┌───────────────┘
                      ↓ ↓
              ┌───────────────┐
              │   REDUCER     │  ← fonction PURE (pas d'effets de bord)
              │               │    switch(action.type) { ... }
              └───────┬───────┘
                      │
                      ↓
              Nouvel état calculé
              { ateliers: [...mis à jour],
                utilisateur: {...mis à jour} }

  DISPATCH = envoyer une action au reducer.
  React appelle le reducer, récupère le nouvel état,
  et re-affiche les composants concernés.


  QUAND useReducer plutôt que useState ?

  useState : 1-3 valeurs simples et indépendantes
  useReducer : état avec plusieurs sous-propriétés liées +
               transitions complexes entre les états +
               logique qu'on veut isoler et tester
gestionPanier.js — reducer et actions 
// On définit les noms des actions comme constantes.
// Avantage : si tu fais une faute de frappe (INSCRRIE au lieu de INSCRIRE),
// JavaScript lève une erreur plutôt que d'ignorer silencieusement.
export const ACTIONS = {
  INSCRIRE:    'INSCRIRE',
  DESINSCRIRE: 'DESINSCRIRE',
  REINITIALISER: 'REINITIALISER'
}

// État initial de l'application
export const etatInitial = {
  ateliers:    [],
  nbInscrits:  0,
  chargement:  false
}

// ─── LE REDUCER ──────────────────────────────────────────────
// Un reducer est une fonction PURE :
//   • Mêmes entrées → toujours mêmes sorties
//   • Pas d'appels API, pas de setState, pas de side effects
//   • NE JAMAIS MODIFIER directement l'état (toujours créer un nouveau)
//
// Elle reçoit :
//   • etat   : l'état ACTUEL
//   • action : l'objet envoyé par dispatch (toujours { type, ...données })
//
// Elle retourne : le NOUVEL état
// ─────────────────────────────────────────────────────────────
export function reducteurAteliers(etat, action) {
  switch (action.type) {

    case ACTIONS.INSCRIRE:
      return {
        // ...etat : copie TOUT l'état actuel
        ...etat,
        // On écrase seulement "ateliers" avec la nouvelle version
        ateliers: etat.ateliers.map(a =>
          a.id === action.id
            ? { ...a, estInscrit: true, currentParticipants: a.currentParticipants + 1 }
            : a
        ),
        nbInscrits: etat.nbInscrits + 1
      }

    case ACTIONS.DESINSCRIRE:
      return {
        ...etat,
        ateliers: etat.ateliers.map(a =>
          a.id === action.id
            ? { ...a, estInscrit: false, currentParticipants: a.currentParticipants - 1 }
            : a
        ),
        nbInscrits: etat.nbInscrits - 1
      }

    case ACTIONS.REINITIALISER:
      return etatInitial // on retourne l'état initial

    default:
      // TOUJOURS avoir un default qui retourne l'état inchangé.
      // Si on dispatch une action inconnue, rien ne casse.
      return etat
  }
}
ComposantAvecReducer.jsx — utiliser useReducer 
import { useReducer } from 'react'
import { reducteurAteliers, etatInitial, ACTIONS } from './gestionPanier'

function GestionAteliers() {

  // useReducer(reducer, etatInitial) retourne comme useState :
  //   etat     → l'état actuel (comme "compte" dans useState)
  //   dispatch → la fonction pour envoyer des actions (comme "setCompte")
  //
  // La différence : au lieu de dispatch(nouvelleValeur),
  //                 on fait dispatch({ type: 'NOM_ACTION', ...données })
  const [etat, dispatch] = useReducer(reducteurAteliers, etatInitial)

  const sInscrire = (id) => {
    // dispatch envoie une action au reducer.
    // Le reducer reçoit { type: 'INSCRIRE', id: '...' }
    // et calcule le nouvel état.
    dispatch({ type: ACTIONS.INSCRIRE, id })
  }

  const seDesinscrire = (id) => {
    dispatch({ type: ACTIONS.DESINSCRIRE, id })
  }

  return (
    <div>
      <p>Inscriptions actives : { etat.nbInscrits }</p>
      { etat.ateliers.map(a => (
        <div key={a.id}>
          { a.titre }
          <button onClick={() => sInscrire(a.id)}>S'inscrire</button>
        </div>
      )) }
    </div>
  )
}

Hooks personnalisés

Un hook personnalisé est une fonction qui commence par use et qui regroupe des hooks React pour les réutiliser. C'est simplement de l'extraction de code — comme une fonction utilitaire, mais pour de la logique avec hooks.

Règle : tout ce qui commence par use est traité comme un hook par React et soumis aux règles des hooks (pas dans une condition, pas dans une boucle).

hooks/utiliserFetch.js — hook de fetch réutilisable 
import { useState, useEffect } from 'react'

// ─────────────────────────────────────────────────────────────
// Ce hook regroupe la logique fetch + loading + erreur.
// Avant il fallait réécrire tout ça dans chaque composant.
// Maintenant : 1 ligne dans le composant = toute la logique.
//
// Il REÇOIT : une URL (string)
// Il RETOURNE : { donnees, chargement, erreur }
//               → tout ce dont le composant a besoin
// ─────────────────────────────────────────────────────────────
function utiliserFetch(url) {

  const [donnees,    setDonnees]    = useState(null)
  const [chargement, setChargement] = useState(true)
  const [erreur,     setErreur]     = useState(null)

  useEffect(() => {
    // Si l'URL n'est pas fournie, on ne fait rien.
    if (!url) return

    // On remet le chargement à true à chaque changement d'URL.
    // Comme ça, si on change l'URL (ex: changer de page),
    // le spinner s'affiche pendant le nouveau fetch.
    setChargement(true)
    setErreur(null)

    const executer = async () => {
      try {
        const rep  = await fetch(url)
        // Si le serveur répond avec un code d'erreur (404, 500...),
        // fetch ne lève pas d'erreur. On doit vérifier manuellement.
        if (!rep.ok) throw new Error(`Erreur HTTP ${rep.status}`)
        const data = await rep.json()
        setDonnees(data)
      } catch (e) {
        setErreur(e.message)
      } finally {
        // "finally" s'exécute TOUJOURS, qu'il y ait une erreur ou non.
        // On arrête le chargement dans tous les cas.
        setChargement(false)
      }
    }

    executer()

  }, [url]) // ← se ré-exécute si l'URL change

  // On expose seulement ce dont le composant a besoin.
  // Le composant ne sait pas comment les données sont chargées.
  // Il sait juste qu'il peut les utiliser.
  return { donnees, chargement, erreur }
}

export default utiliserFetch
PageAteliers.jsx — utiliser le hook en 1 ligne 
import utiliserFetch from '../hooks/utiliserFetch'

function PageAteliers() {

  // 1 ligne remplace environ 20 lignes de logique fetch.
  // Le composant n'a plus à gérer le useState, useEffect, async/await.
  // Il se concentre sur l'AFFICHAGE, pas sur la récupération des données.
  const { donnees: ateliers, chargement, erreur } = utiliserFetch('/api/ateliers')
  //           ↑ on renomme "donnees" en "ateliers" pour plus de clarté

  if (chargement) return <p>Chargement...</p>
  if (erreur)     return <p>Erreur : { erreur }</p>

  return (
    <ul>
      { ateliers?.map(a => (
        <li key={a.id}>{ a.titre }</li>
      )) }
    </ul>
  )
}

TypeScript avec React (TSX)

TypeScript est du JavaScript avec des types. Tu précises ce que chaque variable doit contenir. L'éditeur (VS Code) t'avertit alors si tu te trompes avant même d'exécuter le code.

Exemple : sans TypeScript, tu pourrais passer inscrits="vingt" au lieu de inscrits={20} et le bug n'apparaîtrait qu'à l'exécution. Avec TypeScript, l'éditeur souligne l'erreur en rouge immédiatement.

types/index.ts — déclarer les interfaces de l'app 
// ─── TYPES PRIMITIFS ─────────────────────────────────────────
// string  → texte
// number  → nombre (entier ou décimal)
// boolean → true ou false
// null    → pas de valeur
// undefined → pas défini
// void    → fonction qui ne retourne rien
// ─────────────────────────────────────────────────────────────

// ─── UNION TYPE ──────────────────────────────────────────────
// "a | b | c" = peut être l'une de ces valeurs ET RIEN D'AUTRE.
// Si tu essaies d'assigner 'expert', TypeScript lève une erreur.
// C'est bien plus sûr qu'une simple string sans contrainte. ─────
export type NiveauAtelier = 'beginner' | 'intermediate' | 'advanced'
export type StatutAtelier = 'upcoming'   | 'ongoing'       | 'completed'
export type RoleUtilisateur = 'learner' | 'instructor'

// ─── INTERFACE ───────────────────────────────────────────────
// Une interface définit la forme exacte d'un objet.
// Si tu crées un objet Atelier sans "titre" → erreur TypeScript.
// Si tu ajoutes un champ inconnu → erreur TypeScript.
// ─────────────────────────────────────────────────────────────
export interface Instructeur {
  nom:    string
  avatar: string
}

export interface Atelier {
  id:                  string
  titre:               string
  description:         string
  date:                string
  heure:               string
  duree:               number          // en minutes
  niveau:              NiveauAtelier   // utilise notre union type
  categorie:           string
  instructeur:         Instructeur     // utilise l'interface Instructeur
  nomImage:            string
  maxParticipants:     number
  currentParticipants: number
  estInscrit?:         boolean         // "?" = optionnel (peut être undefined)
  statut:              StatutAtelier
  tags:                string[]        // tableau de strings
}

export interface Utilisateur {
  id:               string
  nom:              string
  email:            string
  avatar:           string
  role:             RoleUtilisateur
  ateliersInscrits: string[]         // tableau d'IDs d'ateliers
}

export interface Filtres {
  recherche: string
  categorie: string
  niveau:    string
}

// Interface pour le contexte : décrit ce que le Provider expose
export interface ContexteSkillHubType {
  ateliers:     Atelier[]
  utilisateur:  Utilisateur
  sInscrire:    (id: string) => void   // fonction qui prend string, retourne rien

CarteAtelier.tsx — composant typé en TSX 
import { useNavigate }         from 'react-router-dom'
import type { Atelier }        from '../types'
import { utiliserSkillHub }   from '../contextes/ContexteSkillHub'

// ─── INTERFACE DES PROPS ─────────────────────────────────────
// On déclare les props attendues dans une interface.
// TypeScript vérifiera que le parent passe les bonnes props.
// ─────────────────────────────────────────────────────────────
interface PropsCarteAtelier {
  atelier: Atelier   // on réutilise l'interface Atelier déjà définie
}

function CarteAtelier({ atelier }: PropsCarteAtelier) {
  //                              ↑ on type les paramètres avec l'interface

  const { sInscrire, seDesinscrire } = utiliserSkillHub()
  const naviguer = useNavigate()

  const complet: boolean = atelier.currentParticipants >= atelier.maxParticipants

  // ─── ÉVÉNEMENT TYPÉ ──────────────────────────────────────────
  // Les événements React sont typés selon l'élément déclencheur :
  //   React.MouseEvent<HTMLButtonElement> → clic sur un <button>
  //   React.ChangeEvent<HTMLInputElement> → onChange sur un <input>
  //   React.FormEvent<HTMLFormElement>    → onSubmit sur un <form>
  // ─────────────────────────────────────────────────────────────
  const gererInscription = (e: React.MouseEvent<HTMLButtonElement>) => {
    e.stopPropagation()
    if (atelier.estInscrit) {
      seDesinscrire(atelier.id)
    } else if (!complet) {
      sInscrire(atelier.id)
    }
  }

  return (
    <div className="carte-atelier">
      <h3>{ atelier.titre }</h3>
      <p>{ atelier.description }</p>

      <button
        onClick={() => naviguer(`/ateliers/${atelier.id}`)}
      >
        Voir les détails
      </button>

      <button
        onClick={gererInscription}
        disabled={!atelier.estInscrit && complet}
      >
        { atelier.estInscrit ? 'Inscrit ✓' : complet ? 'Complet' : "S'inscrire" }
      </button>
    </div>
  )
}

export default CarteAtelier

Formulaires contrôlés

Un formulaire est dit contrôlé quand React est la source de vérité de ses valeurs. Chaque champ est lié à un state. L'input affiche toujours le state, et quand l'utilisateur tape, on met à jour le state. C'est un cycle bidirectionnel.

  LE CYCLE CONTRÔLÉ :

  Utilisateur tape "R"
         │
         ▼
  onChange(e) déclenché
         │
         ▼
  setState("R")     ← React met à jour le state
         │
         ▼
  React re-rend le composant
         │
         ▼
  <input value="R" />   ← l'input affiche le state

  INPUT ←──── value={state}    (React contrôle l'affichage)
  INPUT ─────→ onChange         (l'utilisateur informe React)
FormulaireInscription.jsx — formulaire complet avec validation 
import { useState } from 'react'

function FormulaireInscription() {

  // Tous les champs dans un seul objet state.
  // C'est plus propre que d'avoir un useState par champ.
  const [form, setForm] = useState({
    nom:        '',
    email:      '',
    niveau:     'beginner',
    conditions: false       // boolean pour la checkbox
  })

  // Object pour stocker les messages d'erreur par champ.
  // {} = pas d'erreurs au départ.
  const [erreurs, setErreurs] = useState({})

  // ─── GESTIONNAIRE UNIVERSEL ──────────────────────────────────
  // Une seule fonction pour tous les champs.
  // On utilise e.target.name pour savoir quel champ a changé.
  // Pour les checkboxes, la valeur est dans e.target.checked
  // (pas e.target.value qui vaut toujours "on" pour les checkboxes).
  // ────────────────────────────────────────────────────────────
  const gererChangement = (e) => {
    const { name, value, type, checked } = e.target
    setForm(prev => ({
      ...prev,
      // [name] est une syntaxe de propriété dynamique :
      // si name = 'email', ça devient { ...prev, email: value }
      [name]: type === 'checkbox' ? checked : value
    }))

    // On efface l'erreur du champ dès que l'utilisateur recommence à taper
    if (erreurs[name]) {
      setErreurs(prev => ({ ...prev, [name]: undefined }))
    }
  }

  // ─── VALIDATION ──────────────────────────────────────────────
  // On retourne un objet d'erreurs. S'il est vide → tout est valide.
  // S'il contient des clés → certains champs ont des erreurs.
  const valider = () => {
    const e = {}

    if (!form.nom.trim()) {
      e.nom = 'Le nom est requis'
    }
    if (!form.email.includes('@')) {
      // Validation simple. En production, utiliser une regex ou Zod/Yup.
      e.email = 'L\'email doit contenir un @'
    }
    if (!form.conditions) {
      e.conditions = 'Vous devez accepter les conditions'
    }

    return e // {} si tout est ok, { nom: '...', email: '...' } sinon
  }

  const gererSoumission = (e) => {
    // e.preventDefault() : obligatoire !
    // Sans ça, le navigateur recharge la page lors du submit.
    e.preventDefault()

    const erreursValidation = valider()

    // Object.keys() retourne un tableau des clés de l'objet.
    // Si le tableau n'est pas vide, il y a des erreurs.
    if (Object.keys(erreursValidation).length > 0) {
      setErreurs(erreursValidation)
      return // on s'arrête ici si des erreurs existent
    }

    console.log('Données du formulaire :', form)
    // Ici on enverrait les données à l'API
  }

  return (
    {/* onSubmit sur le <form>, pas sur le button.
         Ça capture aussi la soumission via touche Entrée. */}
    <form onSubmit={gererSoumission}>

      <div>
        <label>Nom</label>
        <input
          name="nom"               {/* name doit correspondre à la clé dans form */}
          value={form.nom}        {/* React contrôle la valeur affichée */}
          onChange={gererChangement}
          placeholder="Votre nom complet"
        />
        {/* Affiche l'erreur seulement si elle existe */}
        { erreurs.nom && <span className="erreur">{ erreurs.nom }</span> }
      </div>

      <div>
        <label>Email</label>
        <input
          type="email"
          name="email"
          value={form.email}
          onChange={gererChangement}
        />
        { erreurs.email && <span className="erreur">{ erreurs.email }</span> }
      </div>

      <div>
        <label>Niveau</label>
        {/* Pour un <select>, value contrôle l'option sélectionnée.
             onChange fonctionne de la même façon qu'un input. */}
        <select name="niveau" value={form.niveau} onChange={gererChangement}>
          <option value="beginner">Débutant</option>
          <option value="intermediate">Intermédiaire</option>
          <option value="advanced">Avancé</option>
        </select>
      </div>

      <div>
        {/* Pour une checkbox : "checked" au lieu de "value" */}
        <input
          type="checkbox"
          name="conditions"
          checked={form.conditions}
          onChange={gererChangement}
        />
        <label>J'accepte les conditions</label>
        { erreurs.conditions && <span className="erreur">{ erreurs.conditions }</span> }
      </div>

      <button type="submit">S'inscrire</button>

    </form>
  )
}

Lazy Loading & Suspense

Par défaut, Vite bundle tout le code en un seul fichier JS. Si ton app a 15 pages, le navigateur télécharge le code de toutes les pages au premier chargement. Avec React.lazy, chaque page n'est téléchargée que quand l'utilisateur la visite.

router.jsx — avec lazy loading sur les pages 
// lazy : transforme un import dynamique en composant React.
// Suspense : affiche un fallback pendant le téléchargement lazy.
import { lazy, Suspense }                    from 'react'
import { createBrowserRouter, RouterProvider } from 'react-router-dom'
import AppLayout                              from './AppLayout'

// ─── IMPORT LAZY ─────────────────────────────────────────────
// import() (sans le mot "import" au début) est un import DYNAMIQUE.
// Il retourne une Promise qui se résout quand le fichier est chargé.
// lazy() accepte cette Promise et crée un composant React.
//
// AVANT : import TableauDeBord from './pages/TableauDeBord'
//         → chargé IMMÉDIATEMENT avec le reste de l'app
//
// APRÈS : const TableauDeBord = lazy(() => import('./pages/TableauDeBord'))
//         → chargé SEULEMENT quand on navigue vers "/"
// ─────────────────────────────────────────────────────────────
const TableauDeBord   = lazy(() => import('./pages/TableauDeBord'))
const TousLesAteliers = lazy(() => import('./pages/TousLesAteliers'))
const DetailAtelier   = lazy(() => import('./pages/DetailAtelier'))
const MesAteliers     = lazy(() => import('./pages/MesAteliers'))
const EspaceFormateur = lazy(() => import('./pages/EspaceFormateur'))

// Composant de chargement affiché pendant le téléchargement
function PageEnChargement() {
  return (
    <div style={{ textAlign: 'center', padding: '4rem' }}>
      Chargement de la page...
    </div>
  )
}

// Helper pour ne pas répéter Suspense partout
const avecSuspense = (Composant) => (
  <Suspense fallback={<PageEnChargement />}>
    <Composant />
  </Suspense>
)
// ↑ fallback : ce qui s'affiche PENDANT le chargement du composant lazy.
//   Dès que le fichier JS est téléchargé et parsé, le fallback
//   est remplacé par le vrai composant.

const routeur = createBrowserRouter([
  {
    path: '/',
    element: <AppLayout />,
    children: [
      { index: true,              element: avecSuspense(TableauDeBord) },
      { path: 'ateliers',         element: avecSuspense(TousLesAteliers) },
      { path: 'ateliers/:id',     element: avecSuspense(DetailAtelier) },
      { path: 'mes-ateliers',     element: avecSuspense(MesAteliers) },
      { path: 'espace-formateur', element: avecSuspense(EspaceFormateur) },
    ]
  }
])

function AppRouter() {
  return <RouterProvider router={routeur} />
}

export default AppRouter

Bonnes pratiques & Architecture

  STRUCTURE COMPLÈTE D'UNE APP REACT BIEN ORGANISÉE :

  src/
  │
  ├── main.jsx              ← Point d'entrée. Crée la racine React,
  │                           importe les styles globaux, monte AppRouter.
  │                           Contient les Providers (contextes).
  │
  ├── router.jsx            ← TOUTES les routes. createBrowserRouter.
  │                           Lazy loading des pages. C'est SON seul rôle.
  │
  ├── AppLayout.jsx         ← Layout partagé. Navigation + <Outlet />.
  │                           S'affiche sur toutes les pages.
  │
  ├── types/
  │   └── index.ts          ← Interfaces et types TypeScript.
  │                           Un seul endroit pour les modifier.
  │
  ├── contextes/
  │   └── ContexteSkillHub.jsx  ← État global. Provider + hook d'accès.
  │                               Logique métier (sInscrire, seDesinscrire...).
  │
  ├── hooks/                ← Hooks personnalisés réutilisables.
  │   ├── utiliserFetch.js      Logique extraite des composants.
  │   └── utiliserLocalStorage.js
  │
  ├── composants/           ← Composants réutilisables.
  │   ├── Navigation.jsx        Un composant = une seule responsabilité.
  │   ├── CarteAtelier.jsx      Pas de logique métier ici.
  │   └── FiltresAteliers.jsx
  │
  ├── pages/                ← Un fichier = une route.
  │   ├── TableauDeBord.jsx     Les pages assemblent les composants.
  │   ├── TousLesAteliers.jsx   Elles ont le droit d'avoir de la logique
  │   ├── DetailAtelier.jsx     (useState, useMemo...) mais délèguent
  │   ├── MesAteliers.jsx       l'affichage aux composants.
  │   └── EspaceFormateur.jsx
  │
  └── données/
      └── mockData.js       ← Données de test. Remplacées par l'API plus tard.


  RÈGLES DES HOOKS — absolument obligatoires :

  ✅ Appeler les hooks au niveau supérieur de la fonction
  ✅ Appeler les hooks seulement dans un composant React
     ou un hook personnalisé (qui commence par "use")

  ❌ if (condition) { useState() }   → interdit dans une condition
  ❌ for (let i ...) { useEffect() } → interdit dans une boucle
  ❌ monBouton.onClick = () => { useRef() } → interdit dans un callback


  TABLEAU RÉCAPITULATIF DES HOOKS :
Hook Rôle en une phrase Exemple d'usage
useState Mémoriser une valeur qui change et met à jour l'écran Compteur, formulaire, toggle menu
useEffect Déclencher du code après le rendu (fetch, timer, event listener) Charger les données API au montage
useContext Lire un contexte global sans passer par les props intermédiaires Accéder à l'utilisateur connecté
useRef Accéder à un élément DOM ou stocker une valeur sans re-rendu Focus sur un input, stocker un timer
useMemo Mémoriser le résultat d'un calcul coûteux Filtrer/trier une grande liste
useCallback Mémoriser une fonction pour stabiliser sa référence Fonction passée à un enfant React.memo
useReducer Gérer un état complexe avec des transitions nommées Panier, formulaire multi-étapes
useNavigate Naviguer vers une URL par code (pas par un lien) Redirect après soumission de formulaire
useParams Lire les segments dynamiques de l'URL (:id) Page de détail d'un atelier
useLocation Lire l'URL actuelle complète (pathname, search, hash) Analytics, breadcrumb dynamique
